# 🔧 修复 GitHub Pages 404 错误

## 问题：访问 https://caleb-lu.github.io/clawguide-dev/ 显示 404

### 解决步骤：

## 步骤 1：启用 GitHub Pages

1. 访问你的仓库设置页面：
   ```
   https://github.com/caleb-lu/clawguide-dev/settings/pages
   ```

2. 在 "Build and deployment" 部分：
   - **Source**: 选择 `GitHub Actions`（重要！）
   - 不要选择 "Deploy from a branch"

3. 点击 **"Save"** 保存

## 步骤 2：检查 Actions 是否在运行

1. 访问 Actions 页面：
   ```
   https://github.com/caleb-lu/clawguide-dev/actions
   ```

2. 应该看到名为 "Deploy to GitHub Pages" 的工作流

3. 点击它查看运行状态

4. 等待完成（通常需要 1-3 分钟）

## 步骤 3：检查构建是否成功

成功的工作流应该显示：
- ✅ 绿色勾选标记
- 状态："Deploy to GitHub Pages"
- 完成时间

如果失败：
- 点击红色 ❌ 查看错误日志
- 通常是因为构建错误

## 步骤 4：等待 CDN 更新

即使 Actions 成功，也可能需要等待几分钟：
- GitHub Pages CDN 更新：1-5 分钟
- 全球节点传播：最多 10 分钟

## 常见错误和修复：

### 错误 1：The workflow is not properly configured
**原因**：GitHub Pages 未设置为使用 GitHub Actions

**修复**：
1. 访问 Settings → Pages
2. Source 选择 `GitHub Actions`
3. 点击 Save

### 错误 2：Build failed
**原因**：代码构建错误

**修复**：
1. 查看 Actions 日志
2. 检查错误信息
3. 在本地运行 `npm run build` 测试

### 错误 3：404 after successful deployment
**原因**：浏览器缓存或 base 配置错误

**修复**：
1. 等待 5-10 分钟
2. 清除浏览器缓存
3. 使用无痕模式访问

## 如果还是不行：

### 选项 A：使用 GitHub Pages 传统方式

如果 GitHub Actions 方式有问题，可以使用传统方式：

1. 修改 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - run: npm ci
      - run: npm run build

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

2. 提交并推送

### 选项 B：手动部署

在本地运行：
```bash
npm run build
# 在 GitHub Settings → Pages → Source 选择 "Deploy from a branch"
# Branch: gh-pages
# Folder: /root
```

然后：
```bash
npm install -g gh-pages
gh-pages -d dist
```
